# RASHIN — Counsel that comes to you.

**RASHIN（羅針）** は、あなた専属の「相談役つき・専門スタッフチーム」を、**会話だけ**で使える仕組みです。
むずかしい操作はいりません。案内役（ナビゲーター）に話しかけると、内容に合った専門スタッフが動いて仕事を片づけます。

> **羅針（らしん）** ＝ 進む方角を示す「羅針盤」。あなたの仕事の向きを見定め、ぴったりの担当へつなぐ案内役です。

## どんな風に動くの？

あなたは案内役と**ふつうの言葉で相談するだけ**。実際の作業は、後ろにいる専門スタッフが担当します。

1. **話しかける** —「この契約書を見て」「この資料を要約して」のように、ふつうに頼む。
2. **案内役が振り分ける** —「これは契約書レビューの仕事だ」と見極め、適任のスタッフに渡す。誰に頼むか考えなくてOK。
3. **専門スタッフが作業する** — 調べる・書く・チェックする。結果は案件ごとの専用フォルダにまとめて保存。
4. **いつでも続きから** —「何を決めたか・どこまで進んだか・次は何か」も記録されるので、後日「あの件の続き」と言えばそこから再開できます。

```
あなた
  │ ふつうの言葉で相談する
  ▼
案内役（羅針）  ── 話を聞いて、依頼の複雑さを見極める
  │
  ├─ 単純な依頼 ──▶  Agent（個別スタッフに委譲）
  │                    ├─▶ 契約書レビュー
  │                    ├─▶ 調べもの（リサーチ）
  │                    ├─▶ 要約
  │                    ├─▶ 校正・推敲
  │                    └─▶ 文章づくり
  │
  └─ 複雑な依頼 ──▶  Workflow（多段自動化）
                       ├─▶ 並行・パイプライン処理
                       ├─▶ 反復改善ループ
                       └─▶ 構造化出力・再開対応

          ↓ どちらの経路も、成果物と記録は…

  「案件フォルダ」にまとまる  →  いつでも続きから再開できる
```

## 最初から使える専門スタッフ

| 担当 | してくれること |
|---|---|
| 契約書レビュー | 日本の法律をふまえて契約書をチェックし、リスクや直すべき点を指摘 |
| 調べもの | テーマをWebで調べ、出典つきで要点を整理 |
| 要約 | 長い文章・議事録・記事を、短い要点にまとめる |
| 校正・推敲 | 日本語の誤字や読みにくさを見つけ、直す提案をする |
| 文章づくり | 箇条書きやメモから、提案書・メール・報告書の下書きを作成 |

「こんな担当もほしい」と思ったら、**あなた専用のスタッフを追加**できます（案内役に頼めば用意してくれます）。

## 仕事は「案件フォルダ」にまとまる

ひとつの仕事（案件）につき、ひとつのフォルダ。
資料・成果物・作業の記録が、ぜんぶその中に入ります。
だから散らからず、**フォルダを開けばその仕事の全体像がわかり、いつでも続きを再開**できます。

## あなたのデータは安全（外に出ません）

- 実際の**案件データ・あなた専用のスタッフ・やり取りの記録**は、**あなたのパソコンの中だけ**に保存されます。
- これらは**インターネットに公開されません**。顧客名や機密が外に漏れない設計です。
- 共有されるのは「仕組み本体」だけで、あなたの中身は共有されません。

## 使い方（3ステップ）

RASHIN は **Claude Code**（Anthropic 社のAIアシスタント）の上で動きます。

1. この一式を手元に用意する。
2. Claude Code で開く。
3. あとは**案内役に話しかけるだけ**。

> はじめての方へ：先に Claude Code の導入が必要です（[公式ガイド](https://docs.claude.com/claude-code)）。導入後は、上の3ステップで使えます。

## もっと詳しく（技術メモ）

エンジニア向けの補足です。ふだんの利用では読まなくて構いません。

- **案内役の動作**は `CLAUDE.md` に定義（会話と差配に専念し、作業はサブエージェントへ委譲）。
- **専門スタッフ＝サブエージェント**の定義: 同梱プリセットは `.claude/agents/presets/`、自作は `.claude/agents/my-agents/`（`my-` 接頭辞・非公開）。
- **設計パターンの活用**: 複雑な依頼では案内役が `docs/agent-patterns.md`（パターン早見表）を参照し、適した進め方（並行化・評価改善ループ等）と実現ツール（`Agent` または `Workflow`）を合わせて提案。必要なときだけ読む軽量カタログなので、ふだんの動作は軽いまま。
- **案件**は `my-projects/<案件名>/`、作業記録は各フォルダの `_journal.md`。案件外のメモは `my-records/`。
- **便利コマンド**: `scripts/new-project.sh`（案件作成）/ `new-agent.sh`（担当追加）/ `now.sh`（信頼できる外部ソースから正確な時刻）/ `update.sh`（OSS版の更新確認）。
- **時刻**は NTP（既定 NICT＝日本標準時）→ 時刻専門 HTTP API（timeapi.io）→ ローカル時計の順で参照。**バージョン**は `VERSION`。
- **公開しない領域**: `my-*`（案件・自作スタッフ・記録）と `.claude/agent-memory/` は `.gitignore` 済み。更新（`update.sh`）でもこれらは消えません。
